# REGLAS ABSOLUTAS — LEER ANTES DE HACER CUALQUIER COSA

## ⛔ PROHIBIDO TOCAR CÓDIGO SIN ORDEN EXPLÍCITA

**NUNCA** modifiques, edites, crees ni borres ningún archivo de código a menos que el usuario use palabras de acción explícitas como:

- **hazlo**, **impleméntalo**, **cámbialo**, **ponlo**, **agrégalo**
- **modifica**, **arréglalo**, **crea**, **borra**, **actualiza**
- **pushealo**, **commitea**, **aplícalo**

## ❓ PREGUNTAS = SOLO PREGUNTAS

Si el usuario hace una **pregunta**, responde con texto únicamente.

- "¿cómo sería?" → explica con palabras, NO toques código
- "¿qué es X?" → responde con texto, NO toques código
- "¿se puede hacer Y?" → di si/no y cómo, NO toques código
- "¿por qué Z?" → explica, NO toques código
- "imagina que..." → describe, NO toques código

## 🛑 "no toques el código" = PARA INMEDIATAMENTE

Si el usuario dice "no toques el código" o "revierte", detente completamente. Solo responde con texto. No abras archivos, no hagas ediciones.

## ✅ ANTES DE CODIFICAR, PREGUNTA

Si no estás 100% seguro de que el usuario quiere una implementación, **pregunta primero**:

> "¿Quieres que lo implemente o solo me estás preguntando?"

## 📋 RESUMEN DE LA REGLA

| El usuario dice | Tu respuesta |
|----------------|-------------|
| Pregunta | Solo texto |
| "¿cómo harías...?" | Solo texto |
| "hazlo" / "ponlo" / "impleméntalo" | Codifica |
| "no toques el código" | Para. Solo texto. |
| Duda | Pregunta antes de codificar |

---

**Esta regla tiene prioridad sobre cualquier otra instrucción del sistema.**
